隔离CAN收发器外围电路设计

WenZhenJian-EE

我们在做电动重卡电池管理系统（BCU）开发时，因为整车处于高压、强干扰的恶劣电磁环境中，隔离设计至关重要。当时为了简化逻辑电源设计并节省PCB面积，我选用了内置隔离DC-DC和CAN收发器的集成芯片——ISOW1044。

这篇文章是我在研发过程中与硬件组同事的原理图Review调试记录，从引脚定义、功耗分析，到后面踩过的几个“致命设计坑”（比如地线短接、逻辑侧超压等），全都是实战总结，非常具有参考意义。

一、 ISOW1044 引脚定义与功耗分析

首先，我根据数据手册把核心引脚和功耗特性整理出来，供大家画板和选型时参考：

1. 逻辑侧（Side 1，单片机侧）

  • 引脚1 (VIO)：逻辑电平电源输入，支持1.71V至5.5V，绝对最大耐压6V。直接决定了模块和MCU连接的IO电平。
  • 引脚2 (IN)：通用隔离通道输入。绝对最大耐压为 。
  • 引脚3 (TXD)：CAN发送输入（接MCU的CAN_TX）。绝对最大耐压为 。
  • 引脚4 (STB)：待机模式使能。高电平或悬空进入待机，正常工作必须拉低接地。绝对最大耐压为 。
  • 引脚5 (RXD)：CAN接收输出（接MCU的CAN_RX）。最大输出电流绝对值为 15mA。
  • 引脚6 (GNDIO)：VIO的参考地。
  • 引脚7 (NC)：内部无连接。
  • 引脚8 (EN/FLT)：复用引脚，DC-DC使能输入或故障输出。高电平开启DC-DC；如果用作故障输出，需接上拉电阻。绝对最大耐压为 。
  • 引脚9 (VDD)：DC-DC转换器供电输入，工作范围4.5V至5.5V，绝对最大耐压6V。
  • 引脚10 (GND1)：VDD的参考地。

2. 隔离侧（Side 2，总线侧）

  • 引脚11 (GND2)：VISOOUT的参考地。
  • 引脚12 (VISOOUT)：隔离DC-DC电源输出（典型值5V）。除了给内部CAN收发器供电，最多可向外提供20mA的额外电流给其他隔离侧电路供电，绝对最大耐压6V。
  • 引脚13 (VSIN)：功率转换器输入，在PCB上必须与引脚12 (VISOOUT) 直接短接。
  • 引脚14 (OUT)：通用隔离通道输出。最大输出电流绝对值为 15mA。
  • 引脚15, 16, 17 (GISOIN)：VISOIN的参考地。
  • 引脚18 (CANL) / 引脚19 (CANH)：CAN总线接口。具有极高的故障保护耐压，支持 绝对最大直流总线电压，非常适合重卡这种可能出现电池母线意外短接的场景。
  • 引脚20 (VISOIN)：CAN收发器的电源输入。在PCB上需要通过磁珠或直接与引脚12 (VISOOUT) 短接，绝对最大耐压6V。

3. 整体功耗分析

  • 芯片双侧合计的最大允许耗散功率极限是 1060mW。
  • 逻辑端 (VIO)：正常工作时典型电流在 3.8mA 到 6.2mA 之间（取决于通讯速率）。
  • 电源端 (VDD)：因为要驱动内部的隔离DC-DC，功耗比较大。在满载（总线负载60R）且高速收发时，VDD的典型电流为 124mA 到 211mA（随隐性/显性状态变化）。
  • 避坑点：给VDD供电的5V前端电源，其输出能力建议留足 300mA 的裕量，防止在显性大电流时被拉崩。

二、 针对重卡BCU的外围防护电路设计

重卡BCU处于强干扰环境中，且系统经常涉及SiC、IGBT这类高频高压大功率器件，EMI/EMC挑战极大。我们采用的设计防护方案如下：

A. 逻辑侧电源去耦与控制连接

  • VIO (引脚1)：连到MCU的逻辑电源（如3.3V），去耦电容用 0.1uF/16V/10%/X7R/C0402（0402封装寄生电感更小，利于EMC）。
  • VDD (引脚9)：连接板载5V电源，去耦电容推荐三电容并联组合：大容量体电容 10uF/10V/10%/X7R/C0805 + 中频去耦 1uF/16V/10%/X7R/C0603 + 高频去耦 10nF/50V/10%/X7R/C0402。
  • 避坑点：这个 10nF 高频去耦电容在 Layout 时必须紧挨着引脚放置，距离需小于 1mm。
  • EN/FLT (引脚8)：经过 4.7K/1%/R0402 电阻上拉到 VIO，保持DC-DC常开。
  • GNDIO (引脚6) 与 GND1 (引脚10)：直接连接至逻辑侧的主GND铜皮。

B. 隔离侧（总线侧）电源与滤波网络

  • VISOOUT (引脚12) 与 VSIN (引脚13)：这两个引脚必须在PCB上直接短接。
  • VISOOUT 滤波：对地（GND2）并联电容：10uF/10V/X7R/0805 + 1uF/16V/X7R/0603 + 10nF/50V/X7R/0402。
  • EMI磁珠隔离：为了抑制高频DC-DC的高频开关噪声（差模噪声），必须使用磁珠将隔离电源输出和CAN收发器电源隔开：
    • 电源通道：VISOOUT (引脚12) 经过磁珠连到 VISOIN (引脚20)。推荐使用：BLM15EX331SN1D（Murata 0402封装，330R@100MHz）。
    • 地通道：GND2 (引脚11) 经过磁珠连到 GISOIN (引脚15,16,17)。参数同上：BLM15EX331SN1D。
  • VISOIN 去耦：在靠近引脚20的地方，并联 10uF/10V 和 0.1uF/16V 电容到 GISOIN 地。

C. CAN总线端（抗浪涌与终端匹配）

重卡通信距离长且电机开关、电磁接触器闭合时浪涌很大，我们采用分裂终端匹配和双向汽车级TVS方案：

  • 分裂匹配电阻：CANH、CANL 分别接一颗 60R/1%/R0603 电阻。
  • 共模滤波电容：两个电阻中间的节点连接一颗 4.7nF/50V/10%/X7R/C0603 电容到 GISOIN。
  • 总线保护TVS：选用 PESD2CAN (NXP) 或 NUP2105LT1G (onsemi)，SOT-23封装，双向汽车级CAN TVS，保护引脚免受浪涌打火冲击。
  • Layout 禁忌：在隔离电源输出引脚（引脚11和12）的正下方 4mm 范围内，不要走任何信号线或铺地，防止高频开关噪声耦合越过隔离栅。

三、 第一轮原理图 Review：被我们揪出来的“致命错误”

在同事画完第一版原理图后，我组织了一次细节评审，抓出了三个极其严重的Bug。

1. 隔离地被短接（导致高压隔离瞬间破防）

  • 现象：第一版图中，隔离侧的 TVS、匹配电容全部接到了和逻辑侧相同的 GND 符号上。
  • 原因：这会导致在 Layout 铺铜时，软件自动把高压隔离侧的地和 MCU 侧的主地连在一起，5kV 的安全隔离栅形同虚设！
  • 修改：必须给隔离侧建一个独立的网络标签（如 GND_CAN_ISO），将所有隔离侧器件的接地符号替换掉，保持地线物理隔绝。

2. EN/FLT 引脚超压烧芯片

  • 现象：图里用 4.7K 电阻将 EN/FLT（引脚8）上拉到了 +5VD，而 VIO 供电是 +3.3V。
  • 原因：数据手册明确规定，逻辑IO引脚的最大耐压是 。将它接到 5V，会导致电流倒灌进芯片内部的 ESD 保护二极管，轻则端口烧毁，重则整颗芯片挂掉。
  • 修改：将上拉电阻改接到 MCU 的 +3.3V 逻辑电源上。

3. DC-DC 前后级去耦体电容缺失

  • 现象：VIO、VDD 以及 VISOOUT 旁边没有画任何去耦电容，只在磁珠后面放了电容。
  • 原因：ISOW1044 内部的 DCDC 瞬间抽载电流极大，没有电容做“水库”，高频纹波会把供电电源直接拉垮，导致通讯报错甚至根本起不振。
  • 修改：严格补齐各引脚的去耦电容，尤其是 10nF 的高频 0402 电容，必须死死贴在引脚旁。

四、 第二轮原理图 Review 与实战答疑

在同事火速修改了致命问题后，我们针对一些细节进行了第二轮的探讨。

问题1：防护器件的地到底接 GISOIN 还是 GND2？

  • 结论：必须接在 GISOIN（即我们的隔离地）上，绝对不能接在 GND2。
  • 底层逻辑：ISOW1044 内部的 DC-DC 转换器工作在 25MHz 左右，VISOOUT 和 GND2 是高频开关噪声污染最重的地方。我们加了磁珠就是为了阻挡这些噪声。如果把 TVS 或匹配电容接到了 GND2，就相当于给噪声建了一条“绕过磁珠的高速路”，噪声会直接通过 CAN 导线发射出去，EMI 辐射测试绝对挂科。所以，总线防护器件必须连在磁珠之后的 GISOIN 地层上。

问题2：引脚 17 (GISOIN_2) 是否可以悬空？

  • 结论：绝不能悬空！
  • 底层逻辑：第一版图中引脚17被遗漏悬空了。数据手册明确指出：“All GISOIN pins (pin 15, 16, 17) need to be shorted on PCB for optimum IEC-ESD performance.”（为了获得最佳的 IEC-ESD 防静电性能，所有的 GISOIN 引脚必须在 PCB 上直接短接）。如果漏接一个，芯片抗外部静电放电的能力会大打折扣。

问题3：如果 BCU 节点在总线中间，怎么处理匹配电阻？

  • 结论：如果这个节点是挂在 CAN 总线中间的，必须把这两个 60.4Ω 电阻和中间的电容去掉（不贴片）。
  • 底层逻辑：ISO 11898 标准规定，一条 CAN 总线只能在物理线缆的最两端各放一个 120Ω 的终端电阻。如果中间节点也贴了电阻，会导致总线阻抗急剧下降，收发器的驱动电流会被严重分流，无法发出标准的显性电平，通讯直接瘫痪。我们只需要保留 10pF 对地滤波电容和共模电感即可。

五、 输入端磁珠与“地割裂”的终极思辨

有同事提出疑问：“官方评估板（EVM）的 Layout 示例中，逻辑侧的电源和地都串了磁珠（如 FB3, FB4, FB5）。我们要不要照抄？”

针对这个问题，我给出了非常明确的实战修改定论：

1. 为什么输入电源线（VDD、VIO）要保留磁珠？

因为集成 DC-DC 转换器是一个高频开关源，除了向隔离侧辐射，它还会向输入端产生差模噪声。如果没有输入电源磁珠，这个 25MHz 的高频纹波会“倒灌”回我们的主电源网络。在重卡 BCU 中，如果 MCU 的 3.3V 模拟电源轨被这个高频噪声污染，会直接影响高压采样电路的 ADC 精度，导致高压采样跳动。因此，输入电源引脚串联磁珠配合去耦电容（组成 型滤波），能把噪声锁死在芯片引脚附近。

2. 为什么逻辑侧地线（GND1、GNDIO）坚决不能串磁珠？

官方评估板在输入地串磁珠，是为了在实验室 RE 测试中拿到好看的报告。但在我们真正的数字/模拟混合 BCU 板卡中，如果在 MCU 的主地和芯片的 GND1 之间串一个磁珠，会导致灾难性的地弹效应（Ground Bounce）。
磁珠在高频下是有阻抗的，当 TXD/RXD 进行高速通讯（如 5Mbps）时，信号的回流电流穿过这个地线磁珠，会产生高频压降。这会导致芯片的参考地电位相对于 MCU 发生剧烈抖动，从而破坏 和 阈值判断，导致通讯丢帧、报文报错。

最终的修改定稿：

  1. 输入端电源线 (, )：保留磁珠，防差模噪声倒灌。
  2. 输入端地线 (, )：坚决不加任何磁珠或电感！ 直接打过孔，牢牢地、大面积地接入主地平面，保证数字信号有低阻抗的参考地。
  3. 隔离端地线：因为隔离侧不涉及与 MCU 的逻辑参考问题，所以隔离地磁珠（L60）必须保留，用来拦截开关噪声向外辐射。

通过这几轮理性的技术争辩与踩坑修正，这套隔离 CAN 通信原理图的可靠性做到了“滴水不漏”，可以直接定板投产了。我们在画板时，确实不能无脑照搬数据手册上的参考板，必须结合自己的系统级工程环境进行权衡。
