失效分析

在做高电压、大功率的电力电子研发时，我们最害怕听到一个词——“单点失效”（SPOF，Single Point of Failure）。

普通的低功耗数字电路要是挂个电阻，大不了芯片死机、重新上电复位；但在处理上百伏甚至上千伏、几十安培电流的高压系统里，一旦关键节点发生单点失效，能量就会瞬间失控。这种失控的电能在微秒级内就能把几十块钱的 SiC MOSFET 炸得粉碎，连带烧毁整块 PCB 铜箔，甚至引发严重的电气火灾。今天我就结合自己在变流器和 DCDC 研发中的避坑经验，来聊聊单点失效的排查、冗余设计的边界，以及怎么揪出那些隐蔽的“共因失效”（CCF）。

我们研发中常见的单点失效

搞硬件的兄弟们在做系统 FMEA 时，下面这几个节点是出了名的“单点失效重灾区”：

发生节点 典型器件示例 失效原理与破坏性后果
辅助供电 DC-DC 辅助电源的开关管或高频变压器 一旦辅助电源挂了，整板的控制芯片、驱动电路、保护比较器瞬间断电。主控 DSP 失去控制权，系统直接“趴窝”或者由于保护失效导致灾难性损毁。
控制核心 DSP 或 MCU 遭遇强电磁干扰、程序跑飞进入死循环、或者时钟停振。如果外围没有设计纯硬件的死区保护，控制引脚卡死在固定电平，极易引发半桥直通，当场炸管。
关键传感 单路 DC-Link 电压采样运放/电阻 如果采样运放坏了，错误地向主控报告母线电压为 0V。主控的闭环 PI 调节器为了拉高电压，会把 PWM 占空比疯狂推到最大，直接导致母线电容过压爆裂或功率管击穿。
散热枢纽 唯一的水冷循环泵或散热风扇 满载工作时散热系统一旦停摆，IGBT 或 SiC 功率模块会在极短时间内因热量无法散发而发生热失控烧毁。

冗余设计的边界在哪？别被“过度设计”反噬

为了搞出高可靠性的工业级或车规级产品，我们在硬件上通常会设计各种防线：

  • 架构冗余：多模块并联，坏了一个，系统自动降额运行，切掉故障模块。或者用双路辅助电源互为备份，母线电压用多路采样交叉校验。
  • 纯硬件保护：千万别把设备的安全全部押注在软件代码上！ 底层的驱动电路必须有独立的退饱和保护（DESAT）、欠压锁定（UVLO）和有源米勒钳位。哪怕 DSP 已经死机得透透的，驱动芯片也能靠硬件比较器在几微秒内强行关断功率管，保住硬件。
  • 失效导向安全 (Fail-Safe)：比如在 PWM 驱动信号线上一定要加下拉电阻。这样万一主控引脚坏了变成高阻态，下拉电阻能确保开关管被稳稳关断，防止栅极电荷积累误导通。

避坑点：冗余不是越叠越好，适可而止才是硬道理

很多兄弟做设计容易走极端，觉得多加几个备份总没错。但是在安规和可靠性工程中，双重冗余基本就是天花板了。比如你把采样供电拆分给两颗独立的 LDO 已经够安全了，如果你还在 LDO 输入端再去套娃备份，那就属于“过度设计”了。

过度冗余会带来三个致命的副作用：

  1. BOM 复杂度反噬 MTBF：你多加一颗芯片，板上就多了一个失效源。每一个元器件都自带故障率（FIT）。层层嵌套冗余，最后往往是因为那些“备份芯片”本身发生虚焊或电容短路，导致系统频繁误报、停机，反而降低了整机的平均无故障工作时间。
  2. 偏离核心目标：我们搞电力电子设备的，和飞在天上的飞机不同。变流设备一旦发现异常，最优解是立刻封锁 PWM、断开交流继电器，安全地趴窝。只要有一路可靠的硬件保护能把输出彻底切断，Fail-Safe 的目标就达到了，没必要像航天器那样搞三模冗余非要维持运行。
  3. 残余风险接受度：安全标准的核心是“容忍单一随机故障”。把关键路径拆成两个独立通道，恰好同时发生随机物理损坏的概率已经是百亿分之一以下了。这种极低概率的事件在工程上被称为“残余风险”，是可以直接接受的。

多重失效与隐蔽的“共因失效 (CCF)”

在做 FMEA 分析时，我们可以忽略两颗完全独立的电阻在同一秒钟发生随机损坏的“巧合”。但是，我们必须严密防范具有连带关系的多重失效：

  • 连锁失效：看起来是多处同时坏，但根子只有一个。比如风扇停转导致温升过高，功率管热击穿，进而引起母线短路炸机。这就要求我们必须在功率管附近加温度传感器，一旦超温强制拉低驱动。
  • 共因失效 (CCF, Common Cause Failure)：这是设计冗余最容易翻车的地方。指的是你特意设计了两个互为备份的防线，结果因为同一个外部诱因被“一锅端”。比如你搞了双路电压采样，但它们用的是同一个偏置电源，或者在 PCB 上紧挨着走线。一旦遇到外部雷击浪涌，两路采样直接同时被击穿损坏。
  • 潜伏故障 + 单点失效：最阴险的双点失效。保护电路（比如过压保护比较器）其实早就虚焊坏了，但因为平时电压正常，系统根本不知道它坏了（潜伏故障）。等到几个月后，系统真的发生过压，这道防线已经没用了，样机当场炸毁。应对办法是必须加上开机自检 (BIST)。

CCF 共因失效排查指南

在研发中，只有做过双重保护或者冗余的系统，才需要去做 CCF 排查。我们要顺着下面这四条物理家谱，把能“一锅端”的隐患彻底切断：

关联维度 共因诱发机制与表现 物理隔离与解耦策略
电源树分布 两路隔离保护芯片的供电来自同一个反激变压器的同组辅助绕组，或共用同一颗 LDO。电源坏了，两路保护一起瘫痪。 将两路冗余系统的供电接在不同的 LDO 或变压器独立绕组上，从源头上切断电源维度的联系。
接地与回流路径 两路保护电路的地（GND）在 PCB 布局上靠得太近，共用了同一个接地孔。大电流浪涌冲过来时，地电平瞬间被抬高，把两个芯片同时烧毁。 严格划分 PCB 地平面，确保两个冗余电路的参考地和回流路径有足够的物理间距，互不干扰。
时钟与逻辑主脑 软件上设计的两套限流、限压保护算法，全都运行在同一个 DSP 里。万一外挂晶振虚焊或者控制核心卡死，两条防线同时崩盘。 在 DSP 外部设计由模拟比较器构成的纯硬件保护电路，完全不依赖软件时钟域，独立关断驱动。
物理与热力学环境 两个互为冗余的功率模块贴在同一个水冷基板上。由于水路堵塞，散热条件恶化，两个模块同时过温烧毁。 将共用散热通道视为共因，引入外围系统级的独立水流/水温监测，一旦异常强行切断主回路。

研发落地三步法

我们在设计一款高可靠性变流器时，要走完这三步分析：

  1. 单点失效地毯式推演 (FMEA)：把 BOM 表里的每一个器件、每一根线都假设短路或开路，分析后果。凡是会导致设备起火、电人的致命单点，必须通过修改设计或者加底层硬件保护来消灭。
  2. 双点失效定点清除：重点排查平时不动作的保护电路。加入开机自检（比如开机时主控发个微弱的脉冲去触发保护，看保护信号能不能反馈回来），把危险的潜伏故障变成可控的“已检测故障”，防患于未然。
  3. 冗余系统逆向排查：针对双路采样、双驱动等冗余节点，从电源、接地、时钟、环境四个维度逆向捋一遍，把所有能导致双防线同时暴毙的单点因素彻底清除。

辩证统一：有了自检，还要排查保护电路本身的单点失效吗？

必须排查！ 自检虽然能把“潜伏故障”转化为系统锁死不开机的“安全状态”，但我们依然要分析保护电路本身的单点失效。
一方面，我们要自证系统在保护电路坏了的时候，能通过自检安全停机（比如比较器坏了，开机自检不通过，主继电器拒绝闭合，保证设备安全）。
另一方面，我们要量化诊断覆盖率，并且防止保护电路设计得过于敏感导致“误报”。如果保护电路天天误报让设备频繁死锁重启，那产品的可用性就直接崩溃了。我们需要在安全性和可用性之间找到完美的平衡点。
