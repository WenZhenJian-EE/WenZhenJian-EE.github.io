隔离型与车载CAN总线硬件设计与调试避坑手记

通信调理与防护：PhotoMOS可控终端电阻、SM712非对称防护、共模电压悬浮死锁与断地烧管物理机制

在工业控制与新能源汽车研发中，CAN 与 RS-485 是最核心的现场总线。然而，许多工程师在硬件设计时，常因为混淆了隔离地（GND_CAN）与系统地的边界、对总线 TVS 寄生电容量与对称性的忽视、或者机械地对待 120 终端电阻，导致通信在台架联调或带载大电流发波时频繁报码甚至烧毁芯片。

今天我们深度拆解可控终端电阻设计、隔离总线对地 RC 泄放、非对称总线 TVS 选型，以及车载系统中最怕遇到的“断地烧芯片”底层物理机制。

一、 软件定义终端电阻：PhotoMOS 隔离切投电路设计

CAN 总线两端节点需要各并联一个 终端电阻以吸收高频反射信号。如果节点处于总线中间，是不允许挂电阻的。为了避免人工拨码的繁琐，在工业网关中，我们设计了软件可控的分裂终端电阻电路。

1. 为什么不能用普通继电器或普通晶体管？

  • 普通继电器：体积大、存在触点弹跳，在振动大的环境中易发生误接触。
  • 普通三极管/MOS管：没有物理隔离，会将总线高频共模干扰直接拉入微控制器地。
  • 最佳方案：使用 PhotoMOS（光控固态继电器，如 OR-860AS-TA1）。它利用内部 LED 光学触发输出端的双向 MOSFET，实现无触点、极长寿命且光电隔离的高速开关。

2. 分裂终端可控电路图设计

  • 工作逻辑：当微控制器 GPIO（CTRL）输出高电平时，NPN 三极管导通，PhotoMOS 内部二极管点亮，输出端触点 3-4 闭合。由两个 精度 1% 的电阻与 电容构成的**分裂终端（Split Termination）**被接入 CAN 总线。
  • 分裂终端的物理意义：电容能将总线高频共模噪声就地滤入隔离地，同时阻抗匹配电阻维持 ，有效抑制高频 EMI 辐射。

二、 TVS 防护选型对比：RS-485 非对称 SM712 与 CAN 24V 容错

总线传输距离长，极易发生空间耦合或物理搭接，因此 TVS 选型是通信接口稳定性最关键的红线。

1. RS-485 为什么必须用非对称 SM712？

RS-485 标准（IT/不接地系统）的共模工作电压规定为 。

  • 如果使用普通的对称 TVS，当总线上出现合法的 共模噪声时，TVS 会强行对地导通泄流，直接导致通信阻断。
  • 如果使用对称 TVS，当线上出现 共模干扰时，TVS 尚未达到开启电压，而大部分 RS-485 芯片在负向能承受的极限仅为 ，导致芯片被击穿。
  • SM712 的非对称机制：其内部集成的二极管结构，实现正向 击穿，负向 击穿，完美贴合 RS-485 芯片的电气耐压边界。

3. 专用防护 TVS 与普通双向 TVS 的物理分野

普通双向 TVS（如普通的 24V 双向贴片管）绝对不能用于高速总线：

  1. 寄生电容量（结电容）：普通 TVS 的结电容高达几百 ，在 速率下会将高速方波信号的沿拖成缓斜坡，报位错误频发。专用 PESD1CAN-UX 结电容仅为 左右。
  2. 两路电容匹配度：差分线两路对地电容必须极其对称（容差 ）。若不对称，高频瞬态下共模干扰会直接在总线内转化为差模干扰，彻底击穿收发器的共模抑制比。

三、 隔离 CAN 的“惹不起躲得起”防悬浮 RC 泄放设计

许多硬件设计以为 CAN 既然是差分信号，那么把信号线拉过去就可以了，隔离芯片侧的地 悬空不用引出。这是灾难性误区。

1. 共模电压溢出死锁

虽然差分放大器只看压差，但芯片内部的差分接收器有最大共模输入电压限制（如 ）。隔离端地如果完全浮空，设备间的分布电容、变压器绕组间电容以及静电摩擦，会使隔离侧地与对端地之间积累数十伏甚至上百伏的压差。共模电位一旦被拉出限制范围，接收器就会进入截止区（死锁），通信完全中断。

2. 防御性设计：高压 RC 阻抗接地法

为了在大功率干扰和静电下维持浮地体系的电位稳定，隔离侧的 必须通过 RC 并联网络接机壳大地（PE）：

  • 大电阻：为静电和浮地漂移提供一条极微弱的直流泄放通路，将共模电压刚性拉回 PE 大地附近，防止电位无限升高击穿隔离芯片。
  • Y电容：隔离电源 DCDC 芯片的开关动作会产生共模噪声，这个电容提供了一个本地的低阻抗高频回流路径，防止通信电缆变成发射天线向外辐射高频 EMI。

四、 非隔离车载 CAN：致命的“断地”烧管物理机制

在非隔离车载网络中，ECU 节点全部共用车辆底盘或车身地（KL31）。

断地故障（Loss of Ground）物理流程

当一个 ECU 节点由于物理振动导致其与车身负极（地线）断开，但电源正极与 CAN_H / CAN_L 依然连通时：

  1. 该 ECU 内部的功率电路、MCU 及各种芯片的供电电流，在失去主地回路后，会寻找一切可能的物理通路回流到负极。
  2. 电平电流会从内部电源轨，穿过 MCU 控制引脚、强行灌入 CAN 收发器内部。
  3. 大电流顺着 CAN_H 和 CAN_L 信号线流向总线网络，最后通过总线其他正常节点的 CAN 收发器内部保护二极管，流回其他节点的车身地。
  4. 后果：CAN 总线电压直接被拉死在 （网络全瘫痪），且该断地收发器由于强行承受了整个 ECU 数百毫安的内部工作电流，会在几十秒内迅速过温热击穿，直接烧毁引脚。
  • 防护：车规级 CAN 收发器（如 TJA1044）通常在芯片内部增加了高阻防倒灌和搭地保护逻辑，能承受大电流冲击，但在布板时主地回路必须设计冗余（双接地销钉），避免单点开路。

五、 NXP TJA 与 TI TCAN 系列选型决策

在目前主流变流器控制板上，TJA 与 TCAN 是两款最核心的竞品：

  • TJA1044 / TCAN1044（8引脚基础款）：管脚完全兼容，支持 CAN FD。TCAN1044 支持更高的 FD 速率（可达 ），总线耐压达 ，ESD 裕量略高于老款 TJA1044。两者在常规控制板上可直接代换。
  • TJA1043 / TCAN1043（14引脚进阶款）：专为车身域控制器或储能主控板设计。具备 **INH（电源控制输出）**和 **WAKE（本地唤醒）**引脚。
    • INH 控制：当系统进入深度休眠时，INH 引脚输出高阻态，直接关断外部 LDO 使得主控 MCU 彻底断电，整板功耗降至微安级。当 CAN 总线上有唤醒报文袭来时，收发器内部有源监听并拉高 INH，重新上电唤醒 MCU。适用于超低待机功耗要求的节点。
