比较器硬件过流保护：迟滞窗口手算与 Layout 避坑手记

在大功率数字电源（比如 2kW 的 LLC 充电模块）或者电机控制器里，硬件过流/过压保护（OCP/OVP）是防止炸管的底线。但很多时候在实验室调试，一跑大电流，保护信号就莫名其妙地误触发或者来回抖动，甚至搞得 MCU 逻辑紊乱。

前阵子调一个 1.2kW 的双向 DCDC 拓扑，就卡在这个问题上整改了半天。今天把高速比较器防噪迟滞窗口的设计模型、MCU 接口的阻抗匹配、还有画板时必须注意的 Layout 避坑细节整理记录下来。

1. 实验室踩坑：过流保护莫名误触发

当时在测一块 DCDC 板子，当输出电流快要接近硬件过流保护（OCP）的阈值时，系统跑着跑着就突然停机报错。

挂上示波器抓波形，发现电流采样信号在阈值附近变动很慢，加上旁边 SiC MOSFET 开关管开关时产生了很大的 空间辐射噪声，杂波直接灌进了比较器的输入脚。

问题所在：原设计里比较器没有做正反馈迟滞。只要输入端的采样电压在阈值附近有哪怕 1mV 的微小波动，比较器输出就会在 0 和 3.3V 之间疯狂抖动，结果 MCU 的中断捕获引脚直接被这堆密集的毛刺信号刷爆，判定为硬件过流保护，强行关了 PWM。

解决办法：必须为比较器引入正反馈，搭一个带“回差（迟滞）”的双阈值窗口。

3. MCU 接口的静态漏电与寄生电容

比较器输出在接进 MCU 保护脚时，有两个电学参数特别容易让人踩坑：

3.1 静态漏电流（Input Leakage）

MCU 芯片的 GPIO 脚即使在空闲时，也是有微弱漏电的，一般上限在 左右。

如果我们在比较器输出和 MCU 引脚之间串了一个特别大的限流电阻（比如 100k），漏电流在电阻上就会产生 的直流压降，直接把逻辑判定电平带偏，或者导致低电平识别不靠谱。

4. Layout 避坑三板斧

在排查实验室干扰时，哪怕你正反馈电阻阻值算得再漂亮，如果 PCB 布局走线太差，一样会翻车：

  1. 正反馈走线贴死：正反馈电阻 必须紧靠比较器的引脚摆放。因为正反馈走线上的电平是突变的（高 ），如果走线拉得很长，它就会像一根天线一样对外辐射高频噪声，甚至和外围的高阻抗节点耦合，导致比较器自激震荡。
  2. 输入与输出物理隔离：比较器的模拟输入走线和数字输出走线绝对不能平行靠近。最好在空间上拉开，或者十字相交，防止输出端跳变的端面通过寄生电容反向注入输入端，产生回馈振荡。
  3. 地平面别被切开：比较器下方的参考地必须保持绝对完整，千万不要让开关管大电流 GND 的走线从比较器参考地正下方穿过，要做好隔离分区。
