MCU相关设计案例

我们在做微控制器系统设计时，经常会遇到各种资源受限、时钟不稳定或者存储选型纠结的问题。今天我整理了几个在实际电路开发中经常碰到的MCU设计案例，包括PWM通道扩展、EEPROM与Flash的底层区别，以及晶振电路设计和不起振的排查心得。

PWM资源不够用？聊聊外部扩展与压控PWM方案

在做多路风扇控制、LED调光或者电机驱动项目时，如果MCU自身的硬件PWM通道被用光了，我们该怎么办？用软件模拟IO翻转不仅精度差，还会榨干CPU的计算资源。我的建议是将PWM生成任务剥离出来，用外部硬件电路来扩展。

实现方案主要有数字控制和模拟控制两大类：

1. 数字控制方案：把占空比交给专用外设或协处理器

这种方案最省心，MCU只需要通过串行总线（I2C或SPI）发送指令，控制外部芯片去维持PWM输出。只有在占空比或频率需要改变时，MCU才通信一次，CPU几乎零负载。

  • I2C接口多通道PWM驱动芯片 (首选 PCA9685)

    • 工作原理：PCA9685 是一款通过I2C总线控制的16通道、12位分辨率的PWM发生器。MCU仅占用SDA和SCL两个引脚，就能独立控制16个通道的占空比。
    • 优点：极度节省引脚，12位分辨率提供4096级占空比调节，技术极其成熟，开源库闭着眼都能用。
    • 缺点：如果只用一两路，单芯片成本略显奢侈。
    • 电路连接：
      • MCU_SCL -> PCA9685_SCL
      • MCU_SDA -> PCA9685_SDA
      • MCU_GND -> PCA9685_GND
      • MCU_VCC -> PCA9685_VCC
      • PCA9685_PWM_Output -> 驱动负载（如MOS管或风扇）

  • 专用风扇控制器芯片

    • 推荐芯片：EMC230x系列、MAX6650/51、AMC6821。
    • 工作原理：除了能用I2C控制占空比，还集成了对风扇测速信号（FG）的读取，可以直接设定目标转速，由芯片在隔离侧自动做闭环控制。
    • 优点：智能温控首选，内置堵转和故障检测，可靠性极高。
    • 缺点：通用性不如PCA9685。

  • 小MCU作协处理器（挂载从机）

    • 推荐芯片：ATtiny85、STM8S003、甚至几毛钱的国产单片机。
    • 工作原理：自己写一个极简的I2C/UART从机固件，主MCU给它发指令，由它去跑自己的定时器PWM。
    • 优点：物料成本低到极致，而且可以自定义各种软启动、呼吸灯等花哨的控制逻辑。
    • 缺点：开发工作量翻倍，你得给两颗芯片编程序，还要多一套烧录治具。
方案 控制方式 优点 缺点 适用场景
I2C PWM驱动芯片 (PCA9685) I2C 简单、成熟、多通道、高精度、资料丰富 成本略高 强烈推荐，几乎所有需要扩展PWM的通用场景
专用风扇控制器 I2C/SMBus 功能专业，集成转速反馈和闭环控制 资料相对少，不通用 需要读取风扇转速或做智能温控的专业场景
小型MCU协处理器 I2C/UART 成本极低，灵活性极高 开发工作量大 DIY或量产对成本极度敏感、有特殊定制逻辑的场景

2. 模拟控制（压控PWM）：用DAC电压换占空比

如果主控芯片有空余的DAC（数模转换器）或者可以通过RC滤波输出模拟电压，我们可以用外部模拟电路来做压控PWM。其核心原理是把周期的三角波或锯齿波与我们的控制直流电压送入比较器进行比较。

  • 经典的555定时器（成本杀手）

    • 工作原理：利用555定时器的控制电压脚（Pin 5, CTRL）。通过给这个脚灌入不同的DAC电压，可以改变内部比较器的阈值。输入电压越高，电容充电时间越长，输出高电平占比就越大。
    • 建议：PWM频率由 决定，风扇控制的话频率建议设计在 15kHz~30kHz，避开人耳能听到的啸叫区。在放电脚串一颗1N4148二极管，能让占空比调节的线性度好看很多。
    • 优点：便宜，结构简单。
    • 缺点：线性度一般，精度易受电容温漂和电阻误差影响。

  • TL494 电源控制器（廉价且稳健）

    • 工作原理：作为老牌的开关电源芯片，它内部带有的锯齿波发生器和误差放大器是天然的压控PWM利器。把DAC输出连到误差放大器输入端，就能输出很稳定的PWM。
    • 优点：性能比555稳，成本同样非常低。

  • 专用压控PWM芯片 (如 LTC6992)

    • 工作原理：高精度三角波发生器与比较器集成，外围只需要几颗电阻调频率，DAC电压直接送入MOD引脚，线性度极佳。
    • 优点：外围省心，指标硬挺。
    • 缺点：贵，比555和TL494贵出一个数量级。

  • 运放 + 比较器自建

    • 工作原理：用运放搭个积分振荡器产生三角波，再过比较器和DAC电压比对。
    • 缺点：元器件多，调试费劲，元器件离散性大会导致每块板子的PWM特性不一致。
方案 实现方式 优点 缺点 适用场景
555定时器 经典时基电路 成本极低，电路简单，易于实现 精度和线性度一般 首选，成本敏感，对精度要求不高的场合
TL494控制器 开关电源控制器 成本极低，性能优于555 配置略复杂 需要比555稍好性能的低成本场合
专用IC (LTC6992) 集成压控PWM芯片 性能好，精度高，线性度佳 成本较高 对控制精度和稳定性有较高要求的专业应用
运放+比较器 从零搭建 灵活性极高，可深度定制 设计复杂，调试困难 学习研究，或有特殊参数需求的场景

存储选型：EEPROM 与 Flash 的底层区别与取舍

我们在做参数存储时，经常在 EEPROM 和 Flash 之间纠结。它们虽然都是掉电保存的非易失存储器，但底层的擦写操作方式完全不同：

  • EEPROM（电可擦除只读存储器）：擦写单位是字节（Byte）。你想改哪个地址的数据，直接写就行，就像铅笔写字一样灵活。
  • Flash（闪存）：擦除单位通常是块/扇区（Block/Sector），写入则是页（Page）。如果你想修改某一个字节，必须先把整个扇区读到MCU的RAM里，把整个扇区擦除掉（全写1），然后把改好的数据重新写入。这就是典型的“读-改-写”机制，非常折腾。

核心参数对比

特性 EEPROM Flash (闪存)
擦写单位 字节 (Byte) 块/扇区 (Block/Sector)
容量 小 (通常几KB到几百KB) 大 (几百KB到几百GB甚至TB)
比特成本 高 低
擦写寿命 极高 (10万 ~ 100万次) 较低 (通常1万 ~ 10万次)
写入速度 单字节写入慢 (ms级) 扇区整体写入快，改单字节极慢
读取速度 较快 极快

我们的选型原则：

  • 如果是频繁修改的设备标定参数、传感器零点、Wi-Fi密码、运行日志，无脑选 EEPROM。
  • 如果是存储大块不常变的数据，比如App固件镜像、大片波形数据、图片字库，选 Flash。

时钟源设计：无源晶振与有源晶振的实战考量

晶振是数字系统的“心脏”，选型和布线直接决定了系统能否稳定起振、跑不跑偏。

无源晶振 (Crystal)

  • 本质：就是一块裸石英晶体片，它本身不具备维持自激振荡的能力，需要MCU内部的振荡放大电路配合。
  • 实战注意：必须在外围精准匹配两颗负载电容。引脚一般为2个（贴片也有4个的，但对角是接地屏蔽脚）。

有源晶振 (Oscillator)

  • 本质：里面除了晶体，还集成了起振、整形、输出的振荡器电路。
  • 实战注意：只需要接VCC和GND，它就能源源不断地吐出高精度的方波时钟信号。一般为4个脚：VCC、GND、CLK_OUT、NC（或Enable控制）。
特性 无源晶振 (晶体谐振器) 有源晶振 (晶体振荡器)
核心元件 仅石英晶体 石英晶体 + 内部振荡电路
工作方式 需外部电路配合 只需供电即可工作
引脚数量 通常2个 通常4个
输出信号 自身不输出信号，是电路的一部分 直接输出稳定的方波时钟信号
精度与稳定性 相对较低，易受外围电路、温度、PCB布局等因素影响 精度高，稳定性好，抗干扰能力强
设计复杂度 较复杂，需精确匹配负载电容 非常简单，可靠性高
成本 低 高

多晶振系统：一个板子放多个晶振会不会乱？

现在的高档SoC或MCU内部都有一个“交通指挥中心”——时钟管理单元（CMU）。它负责管理PLL（锁相环）、分频器、多路选择器等，把不同的外部时钟源分配给CPU、内存控制器、以及各种外设。所以，一个板子上跑多个晶振，非但不会乱，反而是优化功耗与隔离干扰的标准做法。

典型配置如下：

  • 高频主时钟（如25MHz）：通常用一个有源或无源高频晶振，进芯片后经过PLL倍频到几百MHz驱动CPU跑应用。如果在主板上需要把这个时钟分给FPGA或以太网PHY芯片，建议由同一时钟源分发，保证数据同步不会产生相位漂移。
  • 低频实时时钟（32.768kHz RTC）：因为 ，这个频率过15位分频器能极其精准地吐出 1Hz 脉冲，是做日历、倒计时的黄金频率。RTC模块非常省电，即便主供电断了，备用纽扣电池也能推着它走好几年。

RTC 晶振：就算不要断电计时，为什么我也建议你保留？

我们在做工业或者高压电力电子项目时，即使客户说“我们不需要系统时间，断电不需要保存时间”，我也强烈建议大家把 32.768kHz 晶振焊上。

  • 看门狗的最后防线：很多MCU的独立看门狗（IWDG）的时钟源可以配置为使用外部的 32.768kHz。如果你的MCU系统时钟因为强电磁干扰或者PLL死锁导致主时钟停振，看门狗因为用的是独立时钟源，依然能正常超时并强制复位系统。这是我们对抗干扰、保证设备“不死机”的终极防线。
  • 低功耗唤醒：空闲时关掉主时钟和PLL，只留RTC在跑，待机功耗可以从几十mA降到几uA。

晶体谐振器 vs 陶瓷谐振器：不要盲信高频

晶振除了石英材质，还有压电陶瓷材质。两者的差别必须心里有数：

特性 石英晶体振荡器 陶瓷谐振器
核心材质 石英晶体 压电陶瓷材料
精度与误差 极高，常为 较低，常为 级
起振速度 较慢 (ms级) 极快 (us级)
外部电容要求 必须匹配外部负载电容 通常内置负载电容，直接接地即可
成本与空间 成本高，占地方 极其便宜，集成电容，省PCB面积
适用场景 RTC时钟，以太网，高速CAN等需要高精度通讯的场合 流水灯，普通IO扫描等对时钟精度无要求的场合

选型经验：对于大多数MCU，建议首选 8MHz 无源石英晶振。8MHz 是最稳妥的外部频率，MCU内部的驱动器对它匹配最好。芯片可以通过内部倍频推到几十上百兆，对系统性能没影响。不要盲目去买 20MHz 等高频晶振，很多MCU的晶振驱动能力较弱，高频容易导致波形畸变或者干脆不起振。如果必须用高频无源，一定要选要求负载电容小的（如 10pF 左右），降低驱动负担。

晶振的极性与接法：对称晶振与有极性陶瓷谐振器

  • 普通无源晶振：不管两脚还是四脚贴片，都是没有极性的。对于四脚封装，一般只有对角两脚是连接晶片的，另外两个角是接金属屏蔽外壳的，直接拉条线接地即可。
  • 三脚陶瓷谐振器：具有严格的方向性，引脚不能接反！ 以村田三脚谐振器为例：
    • 引脚1：输入端，必须接MCU的 OSC_OUT（MCU的输出驱动晶振的输入）。
    • 引脚2：公共地引脚，必须可靠接地。
    • 引脚3：输出端，必须接MCU的 OSC_IN（将谐振信号反馈回MCU）。

反馈电阻 与阻尼电阻 的配置讲究

在画晶振电路时，你可能见过有人并联电阻，也有人串联电阻：

  • 反馈电阻 （并联在晶振两端）：它的作用是将MCU内部的数字反相器偏置到线性区，强行变成模拟放大器，这是起振的先决条件。现代MCU内部几乎都集成了这个反馈电阻（一般几百KΩ）。所以，我们绝对不要在外部额外并联反馈电阻，否则会破坏原厂偏置点，导致晶振无法起振。
  • 阻尼电阻 （串联在 OSC_OUT 与晶振之间）：作用是限制驱动电流，防止把脆弱的晶体振坏，或者防止产生高频过冲。对于常规 8MHz 晶振，通常不需要，我们在Layout时可以预留一个 0 欧姆电阻的焊盘，调试时发现波形过冲就换成几百欧姆阻尼电阻，没问题就直接直连。

晶振不起振？试试这几招硬件排查大法

如果板子做出来，发现MCU根本不起振，先别慌，按照我下面的实战步骤逐一排查：

  1. 接地回路检查：如果用的是三脚陶瓷谐振器，用万用表测一下中间那个接地引脚，确保它不是悬空或虚焊的。如果地断了，内部电容无法形成回路，肯定不起振。
  2. 引脚交叉检查：核对原理图和Layout，看是不是把 OSC_IN 和 OSC_OUT 给接反了。
  3. 驱动能力与电容不匹配：有些低功耗MCU的驱动跨导设计得非常小，如果你给它配了内置 33pF 电容的陶瓷谐振器，等效负载电容高达 16.5pF，超出了MCU的驱动能力。可以考虑更换为要求 10pF/12pF 的石英晶振。
  4. 软件起振超时调整：有些MCU的库函数中，外部时钟（HSE）起振就绪的超时等待时间（Timeout）设置得太抠搜。如果在某些温度下晶振起振稍微慢了几个毫秒，软件就会误判晶振损坏，直接切回内部RC振荡器。我们在固件里把超时阈值改大一些，往往就能解决这部分边缘问题。
