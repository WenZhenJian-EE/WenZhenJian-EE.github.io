别被仿真骗了：Buck 变换器 CCM/DCM 模式诊断与电压反常升高

我们在实验室用 SPICE 做 Buck 变换器的开环仿真时，很多兄弟会碰到一个诡异的现象：占空比 明明设的是 0.5，输入 是 12V，按照公式 ，输出应该是 6V。但实际仿真出来的电压却高得多，甚至冲到了 7.8V。

这时候别怀疑仿真软件出了 bug，真实的物理世界里，你的电路已经滑向了断续导通模式 (DCM)。

  • CCM 连续模式：输出电压很硬气，只跟输入电压和占空比有关，负载变轻电压也稳如老狗。
  • DCM 断续模式：输出电压会随着负载变轻（负载电阻 变大）、电感量 变小或者开关频率 变低而反常升高，完全脱离了 的控制。

判断系统到底工作在什么状态，核心就是把实际电感量 和临界电感量 做对比：

  • ：系统锁定在 CCM。
  • ：系统滑入 DCM。

CCM 与 DCM：环路控制的“人格分裂”

从控制环路（反馈设计）的角度来看，CCM 和 DCM 的动态特性差了十万八千里。

1. 系统阶数的降维

  • CCM 模式（二阶系统）：
    电感电流永远大于 0，它是一个独立的储能状态。电感 和输出电容 两个储能元件共同发力，系统呈现二阶特性。
    它的伯德图在谐振频率附近有一个高耸的谐振峰（双重极点），相位在此处会直接塌陷 180 度。如果环路补偿没调好，在这里极易发生自激啸叫。
  • DCM 模式（近似一阶系统）：
    因为电感电流在每个周期都会归零，它在低频下失去了作为独立状态变量的资格。电感的作用退化成了一个“受控电流源”，系统动态特性完全由输出电容 和负载电阻 说了算，退化成了一阶 RC 系统。
    它的伯德图非常平滑，没有恶心的谐振峰，相位滞后最多也就 90 度（单极点）。我们在调 DCM 环路时，甚至随便套个简单的 Type II 补偿就能稳住。

2. 动态增益的漂移

  • CCM 的直流增益就是占空比 D，输出电压和负载电流关系不大。
  • DCM 的直流增益会随着负载的变化而疯狂漂移。它的极点频率 ，负载一变，极点位置就跟着动，这要求控制环路必须有足够的鲁棒性。

电感选型指标：为什么推荐感量和临界感量差了 5 倍？

我们在做项目时，经常会遇到这个问题：照着数据手册或者工程软件算出来的推荐电感是 ，但算出来的临界电感 却只有 。这是怎么回事？

这是因为它们遵循了完全不同的设计准则：

1. 基于“纹波率”设计（工程标准做法）

  • 目标：我们要保证输出电压和电流足够平滑，别让纹波把后级芯片搞死。
  • 标准：工程上一般把电感电流纹波系数 限制在 20% 到 40% 之间（即纹波峰峰值是平均电流的 30% 左右）。为了把纹波压得这么小，选出来的电感自然比较大（比如 ）。

2. 基于“临界连续”设计（BCM）

  • 目标：仅仅是为了保住系统别掉进 DCM 模式。
  • 物理直觉：在临界连续模式（BCM）下，电感电流波形刚好是个底边贴着 0A 的三角形。从几何上看，这个三角形的峰峰值（纹波值 ）刚好是平均值（输出电流 ）的两倍。也就是说，此时的纹波系数 。
  • 结论：如果你把 带进电感计算公式，算出来的就是临界感量 （比如 ）。它只是保底线，不是我们正常工作的舒适区。

零损耗的诱惑：Buck 模式切换与软开关本质

在 DCM 模式下，电路其实能实现“部分软开关”：

  • 下管/二极管的零电流关断 (ZCS)：电感电流在周期结束前掉到了 0，二极管自然截止，彻底消灭了二极管反向恢复（Reverse Recovery）损耗，这对效率提升极大。
  • 主开关管（上管）的零电压开通 (ZVS)：能实现多少，完全看输入电压和输出电压的比例。

当电感电流归零、二极管截止后，电感 和 SW 节点的寄生结电容（主要上管下管的 和 PCB 散电容）开始发生谐振。SW 点的电压会以输出电压 为中心发生正弦振荡，理论上最高能荡到 。

软开关实现形式 电压关系 物理动作与效果
完全 ZVS SW 节点能荡到输入电压 ，并冲开主开关管的体二极管进行钳位。此时开启主开关管，就是完美的零电压开通，开关损耗归零。
谷底开关 (Valley Switching) 谐振最高点摸不到 。我们通过控制芯片在谐振电压荡到最高点（即漏源压差最小的谷底）时开通管子，把容性开通损耗降到最低。

高压场景下的主动救赎：强行 FCCM 制造负电流

如果在高压输入、低压输出场景下（比如 24V 转 5V），由于 ，SW 点不管怎么荡都够不着输入电压，主管开通依然是硬开关。

为了彻底消灭开通损耗，我们在现代高频快充（GaN）和服务器供电中引入了强制连续导通模式（FCCM）：

  • 主动制造负电流：在周期后半段，我们故意不关断同步整流下管，让输出电容的能量反向倒灌，在电感里积累一部分反向的“负电流”。
  • 抽干电荷实现 ZVS：当负电流达到设定阈值时，紧急关断下管。电感电流不能突变，这部分负电流只能强行往 SW 节点的寄生电容里灌。电感作为主动源，强行把 SW 电压充到输入电压，冲开上管体二极管钳位。这时候再把上管打开，实现完美的 ZVS。

[!TIP]
设计小窍门：
在 FCCM 软开关电路里，我们反而需要选择比较小的电感。因为根据能量公式 ，在有限的死区时间内，小感量能让反向电流飙得更快，从而积攒足够的能量强行冲开高输入电压。

大电流应用（如 100A CPU 供电）为什么不用软开关？

既然软开关这么好，为什么百安级别的服务器 CPU 电源（VRM）满载时老老实实跑在硬开关的 CCM 模式下？

这就涉及到峰值电流惩罚与导通损耗的平方关系。

开关管的发热损耗公式是 。导通损耗和电流的有效值（平方关系）深度绑定。要想在满载时实现 FCCM 软开关，你就必须把电流纹波拉得极大，让电流谷值掉到 0 以下。

以 100A 负载为例，我们对比一下三种模式的惨烈状况：

工作模式 轻载效率表现 重载 100A 表现
CCM 模式 存在硬开关损耗，轻载效率一般。 纹波很小，电流峰值也就 110A。有效值电流低，发热平稳，器件应力小。
纯 DCM 模式 效率尚可，波形断续。 为了凑够 100A 平均电流，峰值必须飙到 250A 以上，电感直接饱和，MOSFET 当场炸裂。
FCCM 软开关 强行造负电流，实现全软开，效率极高。 要想在 100A 下维持谷底为负电流，纹波必须大于 200A！峰值冲到 210A，有效值电流爆表，导通损耗直接翻倍，输出电容被纹波电流直接热爆。

所以，芯片厂商的黄金调控策略是：轻载跑 FCCM/DCM，用谷底开关或主动软开保效率；重载果断切回 CCM，压低峰值电流，靠多相并联来分摊温升。

我们在工程中是怎么抓 ZVS 时机的？

在纳秒级的开关跳变里，用比较器去直接采样几十伏的 SW 尖峰波形，不仅噪声大、易误判，而且器件极贵。

我们在研发中一般用这三种“平替”策略：

  1. 固定死区盲等（Fixed Deadtime）：
    根据管子的结电容 、输入电压 和电感负电流，估算出 SW 电压爬升所需的时间：。我们在控制芯片里设一个死区时间，时间一到，闭着眼睛发驱动信号。
  2. 自适应死区控制（Adaptive Deadtime）：
    控制器内部集成检测电路，实时监测驱动脉冲的边缘或者效率反馈，每个周期微调死区时间，自动去蹭那个效率最高的平衡点。
  3. 体二极管钳位监测：
    避开高压，只监测上管体二极管导通时产生的那一点点负压钳位信号。一旦抓到这个信号，说明电压已经爬到顶了，立马把上管打开。

DCM 自由衰减振荡的能量来自哪里？

在 DCM 下，电感电流归零后，SW 点会有一长串阻尼正弦衰减振荡（Ringing）。

很多人会问：“这时候电感电流都是 0 了，电感里没能量了，这振荡能量是从哪儿冒出来的？”

其实，这部分能量是输出电压倒灌出来的。当电感电流归零、二极管关断时，电感两端电压其实是 。输出端的能量顺着电感倒灌回来，给 SW 节点的寄生电容充电，从而激发起谐振。电感在这里起到了一个“能量倒手”的通道作用。

根据能量守恒，电容理论上能冲到两倍的输出电压 。在实际电路里，因为有走线电阻，振荡会慢慢衰减。如果我们选大电感，振荡电流峰值 会变小，电阻发热损耗小，振荡波形反而会荡得更持久、更逼近两倍压上限。

动态不平衡：RHPZ 的微观本质与控制死局

做控制的兄弟对“右半平面零点（RHPZ）”可以说是深恶痛绝。

首先，我们要明确一个第一性原理：伏秒平衡定律仅在系统稳定（稳态）时成立。在负载突变的动态过渡期，伏秒平衡必须被打破。正是因为充电伏秒大于放磁伏秒，电感电流的平均值才有动力往上爬。

但是在定频 CCM 下，RHPZ 会把这个调整过程变成一场灾难。

1. RHPZ 是怎么在微观上发生的？

以 Boost 或 Buck-Boost 为例，它们的开关周期 被主时钟死死限死。

这种在动态换挡瞬间，系统必须“往后退一步才能往前迈一步”的欲扬先抑现象，就是 RHPZ 的微观本质。

2. 恒定负载下的 RHPZ 自激振荡

如果我们在绝对干净的恒定负载下工作，RHPZ 会老老实实呆着。但实际板子上充满了各种开关噪声、电源纹波和温度漂移。
如果你的控制环路反应太灵敏，把环路带宽调得太高，跨过了 RHPZ 的固有频率。当噪声引起电压微幅下跌时，环路会疯狂加大占空比，结果触发 RHPZ 效应，电压进一步下跌。控制器一看跌得更厉害了，继续疯狂加占空比，系统瞬间陷入恶性循环，直接在恒定负载下激发出恐怖的低频自激振荡。

所以，设计控制带宽时，必须死死压在 RHPZ 频率的 1/3 以下。

临界连续模式（BCM）的变压器感量设计流程

我们在从零设计一款反激电源时，可以根据满载边界来推导临界电感 ，从而决定系统在什么点切入 CCM 或 DCM。

设计抉择：

  • 如果你想做 DCM：选实际原边电感 。电感存能少，充放电极快，周期里必然会留出死区时间，换来无 RHPZ、好调环路的好处。
  • 如果你想做 CCM：选实际原边电感 。电感是个慢性子，在规定周期内放不完能量，电流只能叠在下一个周期，强行把底座垫高，换来低发热、低峰值电流的好处。

CCM 模式负载突变时的三个环路阶段

最后，我们用时域的微观视角，看看 CCM 电源在面对负载突然加倍（如 1A 突变到 2A）时，环路是怎么调节自洽的：

  1. 稳态平稳运行阶段：
    输入 24V，输出 12V。伏秒平衡死死掐着占空比在 50%。电感充电 5µs，放电 5µs，充电和放电的伏秒积完全相等。电感电流首尾对齐，平均值刚好等于负载电流 1A。占空比像雕像一样动都不动。
  2. 动态打破平衡阶段：
    负载电流突然飙到 2A。输出电容里的存粮被瞬间抽走，输出电压开始下滑。控制环路捕捉到跌幅，急忙把占空比拉大到 55%（充电 5.5µs，放电被迫压缩到 4.5µs）。在这几个开关周期里，充电伏秒大于放磁伏秒，伏秒平衡被打破。没放完的能量被迫留在电感里，电感电流的底座像踩楼梯一样逐周期往上爬，拼命补充负载缺口。
  3. 重新收敛阶段：
    经过几十个周期的调整，电感电流底座终于爬到了全新高度，平均电流稳稳满足了 2A 负载。电压停止下跌并回弹。控制环路发现误差消除了，慢慢收回占空比，重新降回到 50%。充电和放电时间恢复为 5µs。系统在全新的 2A 高电流底座上，重新实现了完美的伏秒平衡。
